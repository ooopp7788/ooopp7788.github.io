hello HEXO!

发表于 2016-10-31   |   分类于 bolg

刚在githuub上建立了我的Hexo ，把HEXO说明文档也留在这。下面是我的搭建流程。

一、hexo使用（首先要在node，git环境下）

安装hexo

可以输入 hexo 命令验证是否安装成功

hexo常用命令

hexo init

初始化你的hexo文件目录结构，生成的MyHexo文件夹下有我们所需的所有文件和目录，其中public为发布目录，themes为主题目录，source\/_post 为发布文章的目录，可以发现里面有一个HelloWorld.md的文件。

hexo g

生成静态文件到public目录下

hexo s

本地发布hexo，输入 http://localhost:4000 可直接本地看hexo的效果

二、在github上发布hexo

github上建立new repository

命名为你的 github主页名字 + github.io，如我的：ooopp7788.github.io
因为只有这个命名下的repository，github会默认发布为你的主页，可以通过 你的github主页名字 + github.io直接访问

将pubilc文件夹remote到new repository

进入MyHexo/public文件夹

初始化git本地仓库

git remote add origin + 刚才建立的new repository地址

然后就ok啦

三、更新hexo和发布流程

新建hexo文章

会在MyHexo/source/_post文件夹下生成一个newPost.md的文件，这个就是我们要用markdown语法写的blog文章。

Hexo生成静态文件

hexo会将我们写好的文章自动生成静态文件到public文件夹中，以用于发布

push到你的github主页

进入MyHexo/public文件夹

使用git命令推送到你的github主页：
1、添加跟踪

2、提交改动，’’ 中是我们为更新添加的说明

3、push到远程分支，默认为origin master
